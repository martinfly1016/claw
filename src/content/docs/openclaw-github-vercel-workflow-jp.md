# OpenClaw + GitHub + Vercel 実戦ワークフロー

このページでは、OpenClawを使って「実装→コミット→デプロイ」までを一気通貫で回す実務フローを示します。

## ゴール
- GitHubでコード管理
- Vercelで自動デプロイ
- OpenClawで更新サイクルを高速化

## 手順
1. GitHubリポジトリ作成
2. Vercelに連携
3. OpenClawでページ修正
4. pushで自動デプロイ

## 推奨ブランチ戦略
- `main`: 本番
- `feat/*`: 機能追加
- `fix/*`: 不具合修正

## 運用KPI
- デプロイ成功率
- 平均リードタイム
- 変更後バグ件数

## 実践Tips
- PR本文に「目的・変更点・影響範囲」を固定テンプレ化
- 変更ごとにスクリーンショットを残す
- 本番反映前にモバイル表示を必ず確認
