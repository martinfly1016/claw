# OpenClaw とは？向いている業務・向いていない業務（実務詳細版）

OpenClaw は、単なる「AI との対話ツール」ではありません。定義された指示に基づき、各種ツール（スキル）を駆使して実際のタスクを完遂する **「業務実行型の自律 AI エージェント」** です。

## 1. OpenClaw の本質
ChatGPT などのチャットツールが「答えを出す」ことを目的とするのに対し、OpenClaw は「ブラウザを開く」「ファイルを書き換える」「メールを送る」といった **具体的アクションを実行する** ことに特化しています。

## 2. 向いている業務（Strong Suites）
1. **反復的な情報収集と要約**
   - 毎日特定サイトを巡回し、新着情報を要約して通知する。
2. **システム間のブリッジ（データ転送・加工）**
   - 受信したメールの内容を解析し、特定のフォーマットで GitHub の Issue や Google Drive に保存する。
3. **継続的な改善ループ**
   - 実行ログを自己分析し、次回の成功率を高めるためのプロンプト修正を自律的に行う。

## 3. 向いていない業務（Weaknesses）
1. **完全な新規創造**
   - 何の制約もない状態から、全く新しいビジネスプランをゼロから生み出す（対話型 AI の方が得意）。
2. **極めて高い正確性が求められる一発勝負**
   - AI の「揺らぎ」が許されない、人間の最終的な署名や法的な確約が必要な瞬間。
3. **権限管理が未整備な環境での操作**
   - 削除権限などの強力な権限を AI にそのまま預けると、誤操作による重大なデータ損失リスクがあります。

## 4. 導入判断の 3 つの基準
自社の業務に導入すべきかどうかは、以下の 3 点で判断してください。
1. **再現性**: その業務は「手順化」されていますか？（AI は手順があるほど強い）
2. **測定可能性**: その業務の「成功」と「失敗」は明確に定義されていますか？
3. **フェイルセーフ**: 万が一 AI が間違ったときに、人間がリカバリできる余地はありますか？

## 5. 導入による KPI の変化
- **人的リソースの解放**: 月間 50 時間の単純作業を 5 時間（AI の確認時間のみ）に短縮。
- **処理速度の向上**: 24時間 365日、リクエストが発生した瞬間に処理を開始。
- **標準化**: 属人化していた判断基準を AI の指示書（Prompt）として資産化。

## 6. FAQ
- **Q: 専門的なプログラミング知識は必要ですか？**
  - A: 基本的な設定には CLI（コマンドライン）の知識が必要ですが、日々の運用指示は日本語の自然言語で行えます。
- **Q: 導入コストはどれくらいですか？**
  - A: OSS（オープンソース）のためソフトウェア自体は無料です。使用する AI モデル（OpenAI 等）の API 利用料と、実行環境（PC や VPS）のコストのみで始められます。
