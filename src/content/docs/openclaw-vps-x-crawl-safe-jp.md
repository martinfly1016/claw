# OpenClaw + VPS でX情報収集を安全運用する方法

X系の情報収集は需要が高い一方、ログイン管理や凍結リスクを誤ると運用事故になります。

## 推奨方針
1. 公式APIまたは許可された手段を優先
2. セッション管理を分離
3. 収集目的を明確化し最小化

## 実務フロー
- 収集対象定義
- 取得頻度定義（過剰取得を防止）
- 異常時停止条件（rate-limit/認証失敗）
- 監査ログ保存

## 技術チェック
- VPS上の時刻同期
- 例外時のリトライ制御
- APIキー失効時の自動停止

## よくある失敗
- 本番キーを開発用途で使い回す
- 取得対象が増えすぎてコスト爆発
- ログに機密が残る

## まとめ
X収集は“速さ”より“安全な継続性”。まずは低頻度・小範囲で始めるのが正解です。
