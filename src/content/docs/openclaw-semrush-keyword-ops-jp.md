# OpenClaw × SEMrush キーワード運用ガイド（日本市場実務版）

このページは、Claw-Navi の実運用で使う **SEMrush 60% + X 40%** の優先順位モデルを、
実際に回せる手順としてまとめたものです。

## 1. 目的
- なんとなく記事を増やすのではなく、検索需要とSNS熱量を同時に取り込む
- 3日サイクルで継続更新し、検索流入を積み上げる

## 2. 3日サイクル（固定）
### Day 1: 収集
- SEMrush で日本DBの関連キーワードを取得
- 指標: Volume / KD / Intent

### Day 2: 優先度付け
- スコア = SEMrush(60) + Xシグナル(40)
- 上位5〜10キーワードを公開候補に確定

### Day 3: 公開・反映
- 新規記事公開 or 既存記事強化
- ホーム最新導線と内部リンクを更新

## 3. キーワード評価の実務基準
- P1: Volume高 + KD中以下 + 商業/課題解決意図
- P2: Volume中 + クラスタ補完
- P3: ブランド名・補助語・後回し語

## 4. 失敗しやすいポイント
1. 単語だけ見て意図を見ない
2. KD低だけで選んで質が薄くなる
3. 公開後に内链更新をしない

## 5. 追跡KPI
- 7日: インデックス数 / 表示回数
- 14日: 主要キーワード順位変化
- 30日: 自然流入・CTR・滞在時間
