# OpenClaw WeChat 連携ガイド（実務強化版）

WeChat連携は通知導線の強化に有効ですが、設計を誤ると誤送信・情報漏洩・障害連鎖のリスクが上がります。
このページでは、検証環境から本番運用までの安全な進め方を解説します。

## 対象
- チーム通知を WeChat に集約したい運用担当
- OpenClaw のチャネル拡張を検討中の開発者

## 導入前チェック
- [ ] 送信対象（誰に何を送るか）を定義
- [ ] 監査ログ保存先を決定
- [ ] 本番キーと検証キーを分離

## 導入ステップ
1. 連携要件を決める（通知種別・頻度・優先度）
2. 検証環境に接続クライアントを導入
3. テストイベントを流して期待通りに通知されるか確認
4. 失敗時の停止条件（API失敗率、遅延、認証失敗）を設定
5. 本番反映

## よくある失敗
### 1) 通知が多すぎて運用崩壊
- 原因: フィルタ設計不足
- 対策: 優先度別チャンネルに分割、閾値で抑制

### 2) 誤送信
- 原因: 宛先ルーティングの不備
- 対策: 本番前に宛先マップを固定化

### 3) 障害時に止まらない
- 原因: サーキットブレーカ不在
- 対策: エラー率閾値で自動停止

## 安全運用ルール
- 最小権限で開始
- キーは環境変数管理
- 監査ログを90日以上保存
- 週次で通知品質レビュー

## KPI
- 通知到達率
- 平均遅延
- 誤送信件数
- 障害復旧時間（MTTR）
